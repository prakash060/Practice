## What they do

### `frontend.yml`
- Builds the frontend from pk-foodfactory
- Deploys dist to S3
- Invalidates CloudFront cache

### `backend.yml`
- Installs backend dependencies in pk-foodfactory-backend
- Packages the backend source
- Uploads the build bundle to S3
- Creates an Elastic Beanstalk application version
- Updates the Elastic Beanstalk environment

## Required GitHub secrets

### Frontend
- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_REGION`
- `AWS_S3_BUCKET_FRONTEND`
- `AWS_CLOUDFRONT_DISTRIBUTION_ID`
- `VITE_API_URL` — HTTPS API CloudFront base, e.g. `https://d3cvs28uau3ofy.cloudfront.net/api` (not http:// Elastic Beanstalk; avoids mixed content)

Custom domain (www.svifoods.com): see DEPLOYMENT-PRODUCTION.md and CUSTOM-DOMAIN-CHECKLIST.md (ACM us-east-1 + registrar DNS; no workflow change).

### Backend
